<br>
Emulation of large-scaling network is known to be time- and resource-consuming,
because each router image has to be booted from the container.
In fact, router images in emulation does not need a full-fledged container,
but only its control plane behaviors.
By taking over the container data plane, virtual device creation and other steps,
REAL is able to perform emulation with significantly less time and memory,
as well as enabling topology-aware scheduling for better cache affinity.
