<br>
Protocol stacks are notoriously difficult to implement and implement correctly.
Yet an application developer might only care about how to configure the transport layer, 
e.g. choosing protocol parsers, extend packet headers,
instead of implementing the bulk logic.
To this end, a domain-specific language (DSL) is proposed to ease transport layer development,
saving tens of thousands of lines-of-code (LoC).
Further, when chained together,
middleboxes under the same DSL can run within the same process,
with protocol stack logic executed only once.
