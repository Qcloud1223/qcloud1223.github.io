<br>
Feature-rich and complex applications face the same problem of excessive instruction cache (I-cache) footage,
so are complex data plane applications (CDPA).
Yet CDPA shows a unique execution pattern to be exploited.
That is, they repetitively receive, process and then send out packets,
which might share processing paths,
but faces I-cache thrashing because the individual path is too long.
We solve this problem by partitioning CDPA code into I-cache friendly stages,
and enabling a batch of packets to go through all stages in a pipelining manner.
